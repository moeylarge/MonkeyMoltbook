- Persisted author risk scoring and live query-time scoring were out of sync during this session. The root causes were fixed in stages:
  1. ingest-time persistence had still been using thin author-row scoring instead of the richer evidence-aware model,
  2. search-time suspicious-author persistence writes were not awaited in a serverless request path,
  3. evidence-backed authors often lacked the stored DB `authors.id` and only had `source_author_id`, so their correct live risk never persisted,
  4. audit hydration initially mapped against in-memory/source-ID style data instead of stored author-row IDs.
- Those persistence/audit issues are now structurally fixed: evidence-backed suspicious-author rows can now be written durably to `entity_risk_scores`, resolved to real stored author IDs, and returned by a dedicated audit endpoint with readable names.
- New dedicated audit lanes now exist in the server:
  - `/api/moltbook/audit/suspicious-authors` → evidence-backed suspicious authors only
  - `/api/moltbook/audit/mint-authors` → separate mint/ticker-spam abuse lane
