# USER_JOURNEY.md

## User motivation
The user wants more matches and less uncertainty about which photos help or hurt their profile.

## Emotional journey

### Stage 1: Curiosity
"I need to know which photos are helping me and which are killing my profile."

### Stage 2: Trust
Onboarding makes the product feel premium and clear, not gimmicky.

### Stage 3: Commitment
Uploading multiple photos feels easy and intentional.

### Stage 4: Tension
During analysis, user anticipates judgment and insight.

### Stage 5: Reveal
Results give validation plus challenge:
- one or two photos clearly help
- one or more photos clearly drag performance down
- there is a path to improvement

### Stage 6: Desire
The user wants deeper detail and a stronger next attempt, creating premium desire and retry momentum.

## Return journey
The user reopens saved analyses, uploads better photos, and tries to improve score/profile strength.

## UX rules from journey
- value must be obvious in seconds
- upload must feel frictionless
- analysis reveal must feel meaningful
- feedback must be direct and useful
- premium must feel like a logical next step, not a hard sell
