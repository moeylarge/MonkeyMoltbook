# RizzMaxx — App Store Screenshot Checklist

## Objective
Finish a complete submission-ready screenshot package with:
- 7 iPhone screenshots
- 7 iPad screenshots

This checklist turns the current loose screenshot plan into a concrete capture sequence.

---

## Global screenshot rules
- Use the real app UI, not mocked marketing composites unless explicitly needed later.
- Favor clean hierarchy and immediate clarity over showing every UI element.
- Keep copy direct, sharp, and non-cringe.
- Avoid overcrowded screenshots.
- Every screenshot should answer one clear user question.
- Maintain consistent visual rhythm across the set:
  - same background treatment style
  - same typography system
  - same framing logic
  - same level of contrast and polish

---

# iPhone screenshot set (7 total)

## 1) Onboarding hero
### Goal
Establish the promise in one glance.

### Screen/source
- onboarding opening screen

### Must show
- headline promise
- clean CTA
- strong first impression

### Copy direction
Use copy in the lane of:
- fix the photos costing you matches
- find your strongest lead photo
- cut weak photos fast

### Capture notes
- keep this clean and bold
- this is the hook shot, not a feature dump

### Status
- [ ] final iPhone capture done

## 2) Upload / set-building screen
### Goal
Show that the app evaluates a real dating-photo set, not one selfie.

### Screen/source
- upload screen with loaded sample set

### Must show
- sample photo set loaded
- metrics / count visible
- set-building rule or guidance visible

### Copy direction
Anchor on honesty:
- load the set you would actually post
- don’t hide the weak photos

### Capture notes
- pick a visually strong sample set
- avoid messy or empty-state feeling

### Status
- [ ] final iPhone capture done

---

## 3) Results hero
### Goal
Show the app’s core value clearly.

### Screen/source
- main results view

### Must show
- score
- what the score means
- best photo to lead with
- first photo to remove

### Copy direction
Results should land in this order:
1. what your score means
2. best photo to lead with
3. first photo to remove

### Capture notes
- this is the single most important screenshot after onboarding
- prioritize clarity over showing too much detail

### Status
- [ ] final iPhone capture done

## 4) Saved analyses / history
### Goal
Show that progress and past runs are preserved.

### Screen/source
- saved/history view

### Must show
- saved cards
- useful scan history structure
- enough information density to feel real

### Copy direction
Keep it utility-driven:
- track what improved
- revisit past analyses

### Capture notes
- make sure saved cards look polished and not cluttered

### Status
- [ ] final iPhone capture done

## 5) Compare view
### Goal
Show that users can improve over time, not just get one score.

### Screen/source
- compare screen

### Must show
- latest vs previous cards
- what changed section
- clear improvement/difference framing

### Copy direction
Use language in the lane of:
- see what improved
- compare old vs new
- stop guessing which set is stronger

### Capture notes
- comparison should be easy to read in 1–2 seconds

### Status
- [ ] final iPhone capture done

## 6) Premium / pricing screen

- premium screen / locked premium preview / local unlock surface

- clean value explanation
- locked price points reflected correctly

### Capture notes

## 7) Supporting proof screen
### Goal

### Capture notes
