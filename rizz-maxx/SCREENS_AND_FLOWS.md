# SCREENS_AND_FLOWS.md

## Screen list
1. Welcome / onboarding
2. Upload photos
3. Review set
4. Analysis loading
5. Results
6. Saved analyses
7. Premium unlock
8. Settings/profile

## Primary flow
1. User opens app
2. Lands on onboarding screen with clear promise
3. Starts upload flow
4. Adds 4-10 photos
5. Reorders/removes weak photos before analysis if desired
6. Taps analyze
7. Sees analysis loading state
8. Receives ranked results
9. Reviews best lead photo, weakest photos, strengths, weaknesses, and action plan
10. Encounters premium unlock for deeper report
11. Saves session

## Return flow
1. Open app
2. View saved analyses
3. Re-run with improved photos
4. Compare and improve again

## Screen requirements

### 1. Welcome / onboarding
- headline: clear dating-profile optimization promise
- short support copy
- one strong CTA
- trust/value framing

### 2. Upload photos
- image picker
- thumbnail previews
- reorder
- remove/replace
- upload count guidance (4-10)
- analyze CTA only activates when minimum set is present

### 3. Review set
- final ordered preview
- strong/weak placeholders not shown yet
- confirm analyze action

### 4. Analysis loading
- staged progress messaging
- no fake technical jargon
- should feel premium, fast, and deliberate

### 5. Results
- profile score hero
- recommended lead photo
- ranking from strongest to weakest
- top strengths
- top weaknesses
- practical action plan
- removal suggestions
- replacement suggestions
- premium gate block

### 6. Saved analyses
- card list of prior sessions
- date/time
- score snapshot
- tap to reopen

### 7. Premium unlock
- show deeper value, not generic paywall fluff
- explain what gets unlocked
- clear price/CTA area

### 8. Settings/profile
- basic account/preferences
- restore purchase
- legal/support links

## Navigation rule
Keep navigation narrow: stack-based with minimal tabs. Results and saved analyses are the core recurring surfaces.
