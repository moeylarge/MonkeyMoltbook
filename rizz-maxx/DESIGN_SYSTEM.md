# DESIGN_SYSTEM.md

## Visual direction
Premium-dark mobile product with strong hierarchy, restrained gradients, crisp cards, and bold CTAs.

## Color system
- Background / primary: #0A0A0F
- Surface / card: #14141C
- Elevated surface: #1B1C26
- Border / subtle stroke: rgba(255,255,255,0.08)
- Primary text: #F5F7FB
- Secondary text: #A2A8B8
- Accent violet: #7C5CFF
- Accent blue: #3D8BFF
- Positive: #2DD4BF
- Warning/highlight: #F59E0B
- Negative/weak photo: #FF5D73
- Premium gradient: linear(violet -> blue)

## Typography system
- Display: 32/38 semibold
- H1: 28/34 semibold
- H2: 22/28 semibold
- H3: 18/24 semibold
- Body: 16/22 regular
- Body small: 14/20 regular
- Caption: 12/16 medium
- Numeric score: 48/52 bold

## Spacing system
4, 8, 12, 16, 20, 24, 32, 40
Use 24 as default screen padding.

## Radius system
- Chips: 12
- Inputs/buttons: 16
- Cards: 20
- Modal/sheet: 24

## Shadow/elevation
Use soft elevated cards only on key surfaces. Avoid bright/glassy excess.

## Button system
- Primary: filled premium gradient, high contrast text
- Secondary: tinted surface with border
- Tertiary: text-only
- Destructive: muted dark red surface
Buttons should be full-width for primary actions on mobile.

## Card system
Cards should carry the product. Use for:
- onboarding value block
- upload tiles
- ranked photo rows
- strengths/weaknesses
- premium teaser

## Input system
Minimal, large tap targets, dark surfaces, clear selected states.

## Upload component system
- dashed bordered empty state
- photo thumbnail grid
- reorder affordance
- remove/replace actions
- clear analyze CTA pinned low on screen

## Score/result display system
Visually prioritize:
1. overall score
2. best lead photo
3. ranking snapshot
4. strengths
5. weaknesses
6. action plan
7. premium unlock

## Icon treatment
Simple outline icons with occasional filled accent moments.

## Motion rules
- 150-220ms default transitions
- subtle card fade/slide
- result reveal should feel staged, not noisy
- avoid excessive bounce or game-like motion

## Mobile rules
- one dominant action per screen
- thumb-friendly CTAs
- no dense text walls
- clear empty/loading/error states

## Tone in UI
Confident, sharp, direct, modern. Never cringe, spammy, or fake-scientific.
