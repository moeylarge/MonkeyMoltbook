# LAUNCH_COPY_NOTES.md

## Onboarding direction
Use direct promise language:
- fix the photos costing you matches
- find your strongest lead photo
- cut weak photos fast

## Upload direction
Keep the user anchored on honesty:
- load the set you would actually post
- do not hide the bad photos from the system
- stronger input truth -> stronger output

## Results direction
The result should land in this order:
1. what your score means
2. best photo to lead with
3. first photo to remove
4. what is working
5. what is costing you
6. what to do next

## Tone rule
- direct
- modern
- sharp
- useful
- confident

Avoid:
- cringe seduction language
- fake certainty
- bloated self-help copy
- clinical beauty-science language

## App Store / screenshot note
Best screenshot candidates are:
- onboarding hero promise
- upload set with strong card layout
- result hero + best photo + first photo to remove
- saved history / compare view
